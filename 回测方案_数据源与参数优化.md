# ETF 智能体 — 回测方案（数据源特性 × 参数优化边界）

> 生成日期：2026-06-11  
> 用途：统一本项目的回测口径；评审其他智能体/协作者回测结论是否可信。  
> 生产参数真源：`scoring.py` / `position.py` / `daily_job.py`

---

## 一、两类数据源的本质差异（必须先读）

### 1.1 CSDN 数据集（`骏/` 目录，回测用）

| 维度 | 特点 |
|------|------|
| **位置** | `C:/Users/32872/Desktop/etf智能体/骏/` |
| **主用文件** | `extracted_net/2020~2023/网络新闻量化统计（按自然日）-{year}.xlsx` |
| **原始压缩包** | `网络财经新闻库/网络新闻量化统计（按自然日）.zip` 等 |
| **时间跨度** | **2020-01-01 ~ 2023-12-31**（1461 日历日，已缓存） |
| **粒度** | **股票 × 日 × 情感计数**（Pos/Neu/Neg），**无新闻标题/正文** |
| **情感标签** | 数据集自带 Sentiment（正面/中性/负面计数），非本项目关键词/LLM |
| **映射路径** | 成分股加权聚合 → ETF 情绪分 0~100（`sentiment_aggregator.py` → `data/csdn_scores/csdn_daily_scores.json`） |
| **覆盖 ETF** | 8 只有效：510300/510330/510050/510500/159915/159949/512010；（512880 仅 3 天；510880/518880/159985/159338 缺失或极弱） |
| **与实盘关系** | **不能替代**东方财富文本新闻；只能回答「长窗口、新闻方向权重是否合理」 |

**CSDN 的统计特性（已测）：**

- 情绪分与**当日**收益相关约 **corr≈0.21**（同步解释）
- 与**次日**收益相关约 **corr≈-0.03**（几乎无预测力）
- → 用 CSDN 优化出的「新闻权重」应理解为**情绪暴露上限**，不能直接等同于 AkShare 主题分的预测力

### 1.2 AkShare / 东方财富（实盘 + 短窗口回测）

| 维度 | 特点 |
|------|------|
| **K 线** | `fund_etf_hist_em` → `data/{code}.csv`；已扩展至 **2020-01 ~ 2026-06**（13 只） |
| **实盘新闻** | `news_fetcher.py`：全球财经流、财联社、新浪等；**无历史日期参数**，仅「当前能抓到的列表」 |
| **历史新闻库** | `historical_news.db` + `historical_news_crawler.py`（东财 np350 资讯流 API） |
| **np350 深度** | 约 **1000 页 × 50 条** 后 API **不再往前**；最早约 **2026-02-24**，**无法补 2025 下半年及更早东财流** |
| **cbkjj/cjsd 静态页** | 分页失效/量极小，**不可作长历史主源** |
| **经济日历** | `econ_calendar.py` + `news_aux_fetcher.py`（百度日历）；缓存约 2020~2026，完整性待验 |
| **CCTV** | `news_cctv` 按日回溯，适合补宏观语境，非 ETF 主题主源 |
| **实盘链路** | 关键词筛选 → `news_llm_scorer` 语义分 → fresh/stale 切分（15:00）→ LLM 决策 → 规则层 |

**AkShare 新闻的关键限制：**

1. **不能**像 K 线一样指定 `start_date=2025-07-01` 拉历史列表（接口设计为「最新 feed」）。
2. 东财历史长窗**只能**靠本地 SQLite 爬虫积累；当前可信全量窗口约 **2026-03-02 ~ 2026-04-30（43 交易日）**。
3. 5 月起东财资讯流在 60h 窗口内常为 **0 条**，与 3–4 月**不可比**。

## 二、回测总体原则（防「别的智能体」常见错误）

| 原则 | 正确做法 | 常见错误 |
|------|----------|----------|
| **防未来数据** | 决策日 D 仅用 `date < D` 的 K 线；新闻仅用 D 日 09:30 前 | 用当日收盘/K 线算特征 |
| **新闻时间切分** | 实盘：上一交易日 15:00 后 = fresh | CSDN 无文本时间戳，不能仿 fresh/stale |
| **训练/验证分离** | CSDN：**2020-01~2021-06 训练，2021-07~2023-12 验证** | 全样本取最优 → 过拟合（如 n45 训练高、验证崩） |
| **目标函数** | 主：**验证集 PnL / Sharpe**；辅：最大回撤、空仓率 | 只看训练集总收益 |
| **管道一致** | 应用 `run_news_backtest.py` 或 `daily_job` 同源函数 | 用 `csdn_backtest.py` 简化版结论代表生产 |
| **尺度一致** | CSDN 新闻分 0~100；AkShare 主题 raw ∈ [-0.85,0.85]→主题分 [7.5,92.5] | 把 CSDN 最优 35% 新闻权重直接套 AkShare 而不做 A/B |
| **LLM** | 回测必须 `--llm-cache-only` 或关 LLM | 无缓存调 API → 不可复现且贵 |
| **样本量门槛** | 新闻类参数：**≥40 交易日** 且分训练/验证才采纳方向 | 56 天全样本网格 → 声称「精确最优」 |

## 三、三轨回测架构（本项目唯一认可的分工）

```
┌─────────────────────────────────────────────────────────────────┐
│  Track A — 纯 K 线轨（L1）                                       │
│  数据：data/*.csv 2020~2026                                      │
│  关闭：新闻、LLM、经济日历（或仅记录不优化）                      │
│  优化：六因子权重、RSI 阈值、trend 内部权重、市场评估阈值         │
│  脚本：自写 grid + features/scoring/position（或 extend 后 CSV）   │
└─────────────────────────────────────────────────────────────────┘
                              ↓ 参数冻结
┌─────────────────────────────────────────────────────────────────┐
│  Track B — CSDN 新闻轨（L2-长窗 / 方向）                         │
│  数据：data/csdn_scores/csdn_daily_scores.json                   │
│  关闭：AkShare 新闻、LLM、fresh/stale（CSDN 无文本）              │
│  优化：news/trend/hist/risk 四维权重、SCORE_GATE（K线+新闻门）    │
│  脚本：csdn_backtest.py（须逐步对齐 rank_etfs_short_race）        │
└─────────────────────────────────────────────────────────────────┘
                              ↓ 仅确认「方向」，幅度折中
┌─────────────────────────────────────────────────────────────────┐
│  Track C — AkShare 全链路轨（L2-短窗 / 实盘一致性）               │
│  数据：historical_news.db + econ_calendar_cache + llm_cache      │
│  窗口：2026-03-02 ~ 2026-04-30（43 日，三源齐全）                 │
│  必须：fresh/stale、historical_news_builder、run_news_backtest     │
│  优化：fresh/stale 权重微调、prompt A/B；**不**用大网格搜 48 参数  │
│  脚本：run_news_backtest.py --compare --llm-cache-only             │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  Track D — 实盘前验证（非优化）                                   │
│  daily_job 单日/多日 dry-run；对比 full.json 与回测同日持仓       │
└─────────────────────────────────────────────────────────────────┘
```

**禁止混用：** 用 Track B 的收益率数字直接写进「实盘东财 3–4 月表现」；用 Track C 的 43 天结果外推 CSDN 四年最优权重。

## 四、目标优化参数清单（按轨道）

### 4.1 Track A — 可优化（L1，高置信）

| 参数组 | 当前值 | 搜索建议 | 模块 |
|--------|----------|----------|------|
| 六因子权重 | 动量25 RSI25 MACD15 量比15 趋势10 布林10 | 每因子 ±5%，步长 5% | `scoring.score_stock` |
| RSI 超买/超卖线 | 75 / 25 | 70~80 / 20~30 | `scoring.score_stock` |
| trend 内部权重 | 1d15% 3d30% 5d30% 量比10% | 1d/3d/5d 各 ±5% | `features._calc_short_race_features` |
| 市场评估分段阈值 | -5→15%, -2→15%, -0.5→40%, +0.5→85%, +2→90% | 阈值 ±0.5~1.0 | `position.evaluate_market_regime` |
| 风险扣分系数 | RSI>82, bb>0.9 等 | 仅验证敏感性，不大范围搜 | `features` risk_penalty |

**样本：** 2020-01 ~ 2023-12（CSDN 同期 K 线齐全）+ 2024-04 ~ 2026-06（实盘 K 线）  
**目标函数：** `验证集总 PnL` 或 `验证集 Sharpe`；报告 `最大回撤`、`Calmar`  
**结算：** 开盘买入、收盘卖出；`ETF_AGENT_STRICT_DATA=1`；摩擦 6bp（与 `daily_pnl` 一致）

### 4.2 Track B — 可优化（L2，中置信，仅方向）

| 参数组 | 当前值 | CSDN 网格结论（参考） | 搜索建议 |
|--------|----------|----------------------|----------|
| 新闻合计权重 | fresh25%+stale10%=**35%** | 甜点 **35%~40%**；45% 易过拟合 | news ∈ [0.30, 0.45]，步长 0.05 |
| trend 权重 | **30%** | 与 hist 合计 45%~50% | trend ∈ [0.20, 0.35] |
| hist 权重 | **20%** | 同上 | hist ∈ [0.10, 0.25] |
| risk 权重 | **15%** | 10%~15% | risk ∈ [0.10, 0.15] |
| SCORE_GATE | **50** | 45 过松(94%通过)，55 过严(7%) | 45, 48, 50, 52, 55 |

**注意：** Track B 中 `fresh`/`stale` **不拆分**（CSDN 无时间戳）；合并为单一 `news` 权重。  
**采用规则：** 取**验证集**最优；若训练与验证最优相差 >2 个步长 → 取**验证次优**或训练/验证折中。

### 4.3 Track C — 有限优化（L2-短窗，实盘对齐）

| 参数组 | 是否优化 | 说明 |
|--------|----------|------|
| fresh / stale 比例 | **可微调** | 在 fresh+stale 总权重固定下，如 25/10 vs 30/5；仅 43 日窗口 |
| SCORE_GATE | **验证** | 用东财+LLM 全链路复测 45/50/55，不单独信 CSDN |
| 经济日分级上限 85/75/65 | **不建议搜** | 43 日内触发模式固定，搜了也是噪声 |
| LLM 决策提示词 | **A/B** | 对比 `llm_cache` 新旧 hash，看 stay_cash / score_gate 次数 |
| `news_llm_scorer` prompt | **A/B** | 需重跑语义分或缓存；成本高 |

**命令模板：**

```powershell
cd "c:\Users\32872\Desktop\etf智能体\etf_agent"
$env:ETF_AGENT_STRICT_DATA = "1"
$env:ETF_AGENT_ALLOW_NETWORK = "0"
py -3 run_news_backtest.py --start 2026-03-02 --end 2026-04-30 --sources all --use-llm --llm-cache-only --tag llm
```

**前置条件：** `historical_news.db` 东财流齐全；`data/econ_calendar_cache/`；`data/llm_cache/` 覆盖每个交易日。

### 4.4 当前生产公式（Track C/D 应对齐）

```
fresh_theme_score = clip(50 + fresh_raw×50, 0, 100)  # 经量价确认可打折
stale_theme_score = clip(50 + stale_raw×50, 0, 100)

final_score = fresh×0.25 + stale×0.10 + trend×0.30 + hist×0.20
            - risk×0.15 - rotation_penalty

SCORE_GATE = 50
经济日：high_count → cap 85% / 75% / 65%
单 ETF 硬限：30%
```

## 五、具体回测流程（逐步执行）

### 5.1 Track A 流程

1. `extend_klines.py` 确认 13 只 CSV 覆盖 2020-01-01 起。
2. 对每个交易日 D（跳过周末）：
   - `pool = TRADING_POOL`（进攻池按 OFFENSIVE_ON_THRESHOLD 规则）
   - `rank_etfs_short_race(pool, D)` **但将 theme 权重置 0**（或 theme_signals 全中性 50）
   - `evaluate_market_regime(D)` → `invest_ratio`
   - `allocate_short_race` → 持仓
   - PnL = Σ volume × (close-open)
3. 网格搜索 L1 参数 → 记录训练/验证 PnL。
4. **冻结** L1 最优 → 写入 `scoring.py` / `features.py` / `position.py`。

### 5.2 Track B 流程

1. 确认 `data/csdn_scores/csdn_daily_scores.json`（1461 天）。
2. 训练期 2020-01-01 ~ 2021-06-30；验证期 2021-07-01 ~ 2023-12-31。
3. 对每个 (Wn, Wt, Wh, Wr) 组合：
   - 用 **与生产尽量一致** 的 `rank` 逻辑（建议改造 `csdn_backtest.py` 调用 `rank_etfs_short_race` 并注入 CSDN 主题分，而非内联 `rank_with_csdn`）。
   - 跳过 `adjust_invest_ratio_by_news`（已支持 `_skip_news_adjust`）。
   - 应用 SCORE_GATE、经济日历（若有缓存）、市场评估。
4. 选验证集最优 → **仅调整 fresh+stale 合计权重方向**（如 30%~40%），写入 `scoring.py`。
5. 报告：按 ETF 披露 512880/510880 数据缺失带来的偏差。

### 5.3 Track C 流程

1. 审计 `historical_news.db`：43 日内东财资讯流每日 >0 条。
2. `historical_news_builder.build_historical_signal` 含 **fresh/stale 切分** + 关键词分（**不含** `news_llm_scorer`，除非单独建缓存）。
3. `run_news_backtest.py` 与 `daily_job` 同源 `run_decision`。
4. 对比实验：
   - rule-only vs LLM（`--compare --llm-cache-only`）
   - 权重 ±5% 敏感性（最多 3×3 组合，不做 630 格）
5. 结论表述：**「在 2026 年 3–4 月东财三源齐全窗口内」**，不写「四年验证」。

### 5.4 验收标准（给其他智能体的检查表）

- [ ] 是否写明训练/验证切分？
- [ ] 是否 `date < trade_date` 取 K 线？
- [ ] 东财回测是否限定 2026-03~04？
- [ ] CSDN 回测是否注明 ETF 覆盖缺口？
- [ ] LLM 回测是否 cache-only？
- [ ] 是否区分 `csdn_backtest.py` 与 `run_news_backtest.py`？
- [ ] 优化参数数量是否 <10/轨道（防过拟合）？
- [ ] 是否报告最大回撤而非仅总收益？

## 六、难以回测优化的参数及理由

### 6.1 原则上不可优化（L3 — 设计/合规）

| 参数 | 理由 |
|------|------|
| 单 ETF 仓位硬限 30% | 风控合规，非收益最大化目标 |
| 比赛输出 100 股整数倍 | 赛规约束 |
| 新鲜新闻 > 日历 > 陈旧 > 量价（语义层） | 架构约束；规则层对经济日硬顶限仓 |
| LLM 与规则「取较低仓位」 | 风控哲学，优化会破坏保守性 |

### 6.2 数据不足无法优化（L3 — 样本/触发稀疏）

| 参数 | 理由 |
|------|------|
| 关键词词典 `ETF_THEME_KEYWORDS` | 数百条人工映射；改变一词连锁改变筛选集；CSDN 无文本无法校准 |
| 强/弱信号阈值 0.35/0.12 | 改变阈值改变进入 LLM 的文章集，非独立参数；43 日内触发次数不足以稳定估计 |
| 高位接盘 ×0.35、下行利好 ×0.45 | 56 天内触发 <10 次，PnL 差异 <1% |
| 经济日分级 85/75/65 三档 | 短窗内高影响日几乎天天有，档位无法区分 |
| `adjust_invest_ratio_by_news` 各倍数 0.55~0.65 | 与新闻置信度联动，43 日样本下各档收益接近 |
| 动态闸门 `SCORE_GATE_MODE=dynamic` | 依赖 LLM 非确定性输出；需大量 LLM 缓存 A/B |
| 轮动惩罚 2.5/天、上限 10 | 依赖路径历史，独立网格意义有限 |

### 6.3 管道不一致导致「假可优化」（其他智能体高发）

| 参数 | 为何不能信其回测结论 |
|------|---------------------|
| `news_llm_scorer` 各档分数 | `run_news_backtest` **默认不走**语义打分；若回测未接入则与实盘不一致 |
| fresh/stale 权重 | CSDN Track B **无** fresh/stale；在 CSDN 上搜出来的比例不能代替东财切分 |
| LLM `position_ratio_hint` | `csdn_backtest` 无 LLM；全链路 43 日又太短 |
| 东财 2025H2 新闻权重 | np350 **爬不到**该段历史，任何「2025 下半年东财回测」均不可复现 |

### 6.4 尺度不匹配导致「假最优」

| 问题 | 说明 |
|------|------|
| CSDN 0~100 vs AkShare 主题分 | 同样 35% 权重，数值含义不同；跨源优化必须分轨报告 |
| CSDN 日级聚合 vs 东财 60h 文本窗 | CSDN 是**全日**情绪；实盘是**盘前可用**子集，时间对齐误差不可避免 |
| 缺失 ETF 默认 50 分 | 510880/512880 等在 CSDN 回测中接近中性，**拉低**新闻因子有效性估计 |

## 八、与其他智能体结论的「快速否决」清单

若对方报告出现以下表述，**默认不可信**，除非补齐第三节检查表：

1. 「用东财 API 回测了 2025 年全年」—— np350 无此深度。  
2. 「CSDN 四年回测 +85% 证明实盘可达」—— 管道简化 + ETF 缺失 + 训练过拟合。  
3. 「630 组参数在 56 天全样本最优」—— 未做验证集。  
4. 「回测与实盘完全一致」—— 回测缺 `news_llm_scorer` 与 LLM 语义分（除非显式接入）。  
5. 「新闻权重 60% 为最优」—— 56 天样本下幅度不可信；CSDN 验证集亦显示 45% 过拟合。  
6. 「用当日收盘价做决策」—— 未来函数。
